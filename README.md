# QuickVM

Windows Guest OS 환경에서 가상 머신(VM)을 최적화하기 위한 자동화 스크립트 프로젝트

## 프로젝트 목적

본 프로젝트는 Windows Guest OS 환경에서 VM을 최적화하기 위한 자동화 스크립트 제작을 목적으로 합니다.

## 주요 목표

- **자동화**: OS 설치 후 반복적인 설정 작업 자동화
- **경량화**: 불필요한 서비스 및 기능 제거를 통한 용량 최적화
- **성능 최적화**: VM 환경에 최적화된 시스템 설정 적용

## 주요 기능

QuickVM은 다음과 같은 최적화 기능을 제공합니다:

1. **불필요한 기본 프로그램 제거** - Bloatware, Xbox, OneDrive 등 제거
2. **오른쪽 클릭 메뉴 변경** - Windows 10/11 스타일 선택
3. **전원 최적화** - 고성능 모드 설정, 절전 모드 비활성화
4. **성능 최적화** - 시각 효과 조정, 시작/복구 설정 변경
5. **시스템 및 파일 탐색기 설정** - PC 이름 변경, 탐색기 설정
6. **바탕화면 및 잠금화면 설정** - 배경, 색상, 아이콘 설정
7. **시작 메뉴 및 작업표시줄 설정** - 추천 비활성화, 위젯/작업보기 끄기
8. **시작 프로그램 및 개인정보 설정** - VMFort만 활성화, 프라이버시 설정
9. **정리 및 저장소 최적화** - 임시 파일 정리, 디스크 정리, 이벤트 로그 삭제

## 실행 요구사항

- **관리자 권한 필수**: 모든 스크립트는 관리자 권한으로 실행해야 합니다
- **OS 환경**: Windows 10/11 Guest OS (가상 머신)
- **파일 형식**: `.bat` 배치 파일

## 사용 방법

### 일반 실행 (권장)

1. `QuickVM.bat` 파일을 **마우스 우클릭**
2. **관리자 권한으로 실행** 선택
3. 메뉴에서 원하는 기능 선택 (1-9)
4. 확인 메시지에 `Y` 입력하여 실행

또는 관리자 권한 CMD에서:
```cmd
QuickVM.bat
```

### 도움말 보기

```cmd
QuickVM.bat --help
QuickVM.bat -h
QuickVM.bat /?
```

### 디버그 모드 (로그 기록)

문제 해결이나 상세한 작업 기록이 필요한 경우:

```cmd
QuickVM.bat --debug
```
또는
```cmd
QuickVM.bat -d
```

## 디버그 모드

### 개요
디버그 모드를 활성화하면 모든 작업의 상세한 로그가 파일로 자동 저장됩니다.

### 특징

- **로그 파일 자동 생성**: `result_YYYYMMDD_HHMMSS.txt` 형식
- **저장 위치**: QuickVM.bat 파일과 같은 폴더
- **메인 메뉴 표시**: `[DEBUG MODE ENABLED]` 표시
- **성능 영향 없음**: 일반 모드에서는 로그 오버헤드 전혀 없음

### 로그 레벨

- `[SUCCESS]` - 작업 성공
- `[FAIL]` - 작업 실패 (실패 사유 포함)
- `[INFO]` - 일반 정보 및 사용자 동작
- `[ERROR]` - 치명적 오류

### 로그 예시

```
========================================
 QuickVM - Operation Log [DEBUG MODE]
 Date: 2025-11-26 14:30:25
========================================

[INFO] Started with administrator privileges

========================================
 Menu 1: Remove unnecessary default programs
========================================
[SUCCESS] Removed bloatware apps
          Successfully removed: 35, Failed/Not found: 10
[SUCCESS] OneDrive removed completely

[INFO] Menu 5 skipped by user

[INFO] Program exited by user
========================================
 End of Log
 Completed: 2025-11-26 14:35:12
========================================
```

## 주의사항

- 일부 작업은 시스템 재부팅이 필요합니다
- Menu 9의 정리 작업 중 일부는 되돌릴 수 없습니다 (복원 지점 삭제 등)
- 프로덕션 환경보다는 VM 테스트 환경에 적합합니다
- 디버그 로그 파일은 자동으로 삭제되지 않으므로 주기적으로 정리하세요
